- 배선 연결 구조 확정: 배터리 → 충방전 모듈 → USB-C female 소켓 PCB → 라즈베리파이 DSI 커넥터 → DSI 케이블(S자 고정) → 터치스크린
- 납땜 포인트 2곳 확정: ① 배터리 전선 → 충방전 모듈 Battery +/-, ② 충방전 모듈 Output +/- → USB-C female 소켓 PCB V+/GND
- 케이스 설계 방향 확정: 휴대 방향 가로(닌텐도 스위치형), 3D 프린팅(Fusion 360 또는 OnShape), 측면 통풍구 확보, DSI 케이블 S자 고정(최소 5mm 여유)

- Open WebUI 모델 운용 전략 확정 (8B 응답 완료 후 5분 자동 언로드 또는 수동 전환)

## 2026-06-25 (9일차) - 케이스 설계 확정 및 킬스위치/LED 계획

### 한 일
- 하드웨어 동작 원리 학습 완료
- 케이스 설계 확정 (2층 배치, 포트 배치)
- 킬 스위치 + RGB LED 방식 확정
- 추가 부품 구매 완료/취소

### 학습 내용
- 전원 계통 흐름: 배터리 → IP5305T → USB-C female PCB → Pi5
- IP5305T 역할 및 파워패스 개념
- LiPo 배터리 규격 읽는 법 (605080)
- DSI vs HDMI 차이
- 부품별 단자 입출력 정리

### 주요 결정
- 케이스 2층 배치: IP5305T + USB-C female PCB를 배터리 옆에 수직 배치, U자 어댑터로 Pi5 직결
- 배터리·Pi5 영역 밖으로 삐져나오는 부분까지 케이스로 감쌈
- microSD: 내부 고정, 외부 노출 안 함
- 위쪽 긴 면: micro HDMI ×2, USB-C 전원 노출 / 아래쪽 긴 면: 이더넷, USB-A ×4 노출
- 충전 포트: IP5305T 아래쪽 외부 노출
- 킬 스위치/RGB LED: 측면 공간 비워둠 (부품 도착 후 확정), 전원 버튼은 Pi5 내장 버튼 활용
- 킬 스위치 방식: rfkill 소프트웨어 확정. Wi-Fi ON=파랑, OFF=빨강
- ⚠️ 위 내용 중 microSD 비노출, RGB LED 측면 배치는 기존 design-concept.md/Notion 체크리스트의 가정과 충돌하며 본 항목 기준으로 정정함
- ⚠️ 빨간색이 기존 온도 위험(80도+), 저전압 경고와도 겹쳐 세 가지 의미가 중복됨. 구분 방법은 제시된 자료만으로는 알 수 없어 정하지 않음
