# 프로젝트 설계 컨셉 (raspi-local-ai)

## 한 줄 정의
> 개인정보 유출 걱정 없이 어디서든 AI를 쓸 수 있는, 주머니에 들어가는 프라이버시 우선 단말

## 핵심 원칙
> 기본은 차단, 의도적으로만 열린다

## 프로젝트 동기
- 클라우드 AI 사용 시 개인정보가 외부 서버로 전송되는 문제
- 카톡 등 메신저를 통한 파일 공유 시 데이터 유출 우려
- "내 데이터는 내 기기 안에서만 처리되어야 한다"는 원칙

---

## 운영 모델

### 1차 완성
- 휴대 모드 단독 사용
- 다른 기기와 자료 주고받을 때만 Tailscale 활성화
- 24시간 서버 운영 없음

### 2차 확장
- 거치 시 이더넷 연결 감지 → 서버 모드 자동 전환
- 외부에서 폰/노트북/태블릿으로 집 라즈베리파이 AI 접속

### 케이스 전략
- 1차: 케이스 없이 운영 (GPIO 캡 + 뒷면 아크릴판)
- 2차: 3D 프린팅 케이스 (킬 스위치/LED 장착 시)
- 2차 설계 도구: Fusion 360 또는 OnShape 중 선택
- 휴대 방향: 가로(닌텐도 스위치형), 측면 통풍구 확보
- DSI 케이블: S자 고정 방식, 최소 5mm 여유 확보

### 케이스 레이아웃 확정 (2026-06-25)
- 2층 배치: IP5305T + USB-C female PCB를 배터리 옆에 수직 배치, U자 어댑터로 Pi5 직결
- 배터리·Pi5 영역 밖으로 삐져나오는 부분까지 케이스로 감쌈
- microSD: 내부 고정, 외부 노출 안 함
- 위쪽 긴 면: micro HDMI ×2, USB-C 전원 노출
- 아래쪽 긴 면: 이더넷, USB-A ×4 노출
- 충전 포트: IP5305T 아래쪽 외부 노출
- 전원 버튼: Pi5 내장 버튼 활용, 케이스에 구멍만
- ⚠️ microSD 비노출은 위 "케이스 전략"에 없던 내용이며, 기존 가정(접근 구멍)과 충돌 시 본 항목 기준으로 정정함

### 킬 스위치 + RGB LED 확정 (2026-06-25)
- 방식: rfkill 소프트웨어
- Wi-Fi ON: 파랑, OFF: 빨강
- 위치: 측면 공간 비워둠 (부품 도착 후 확정)
- 추가 구매 필요: 토글 스위치 1개, RGB LED 1개, 저항 3개
- ⚠️ 빨간색이 기존 온도 위험(80도+), 저전압 경고와도 겹쳐 세 가지 의미가 중복됩니다. 구분 방법은 제시된 자료만으로는 알 수 없어 정하지 않음

### 배선 연결 구조
- 배터리 → 충방전 모듈 → USB-C female 소켓 PCB → 라즈베리파이 DSI 커넥터 → DSI 케이블(S자 고정) → 터치스크린
- 커넥터 대신 납땜 직결 방식 유지

### 납땜 포인트
- ① 배터리 전선 → 충방전 모듈 Battery +/-
- ② 충방전 모듈 Output +/- → USB-C female 소켓 PCB V+/GND
