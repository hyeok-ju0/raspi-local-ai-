# 프로젝트 설계 컨셉 (raspi-local-ai)

## 한 줄 정의
> 개인정보 유출 걱정 없이 어디서든 AI를 쓸 수 있는, 주머니에 들어가는 프라이버시 우선 단말

## 핵심 원칙
> 기본은 차단, 의도적으로만 열린다

## 프로젝트 동기
- 클라우드 AI 사용 시 개인정보가 외부 서버로 전송되는 문제
- 카톡 등 메신저를 통한 파일 공유 시 데이터 유출 우려
- "내 데이터는 내 기기 안에서만 처리되어야 한다"는 원칙

---

## 운영 모델

### 1차 완성
- 휴대 모드 단독 사용
- 다른 기기와 자료 주고받을 때만 Tailscale 활성화
- 24시간 서버 운영 없음

### 2차 확장
- 거치 시 이더넷 연결 감지 → 서버 모드 자동 전환
- 외부에서 폰/노트북/태블릿으로 집 라즈베리파이 AI 접속

---

## 하드웨어 설계

### 적층 구조
```
[DSI 3.5인치 터치스크린]  약 7mm  ← 전면
[LiPo 2000mAh 배터리]    약 5mm
[라즈베리파이 5 보드]     약 17mm
[공식 액티브 쿨러]        약 13mm  ← 후면
총 약 42mm
```

### 부품 리스트
| 부품 | 사양 | 구매처 | 상태 |
|---|---|---|---|
| 라즈베리파이 5 8GB | 정품 | 디바이스마트 | ✅ 구매 완료 |
| 공식 액티브 쿨러 | Raspberry Pi Active Cooler | 디바이스마트 | ✅ 구매 완료 |
| 호환 PD 27W 어댑터 | USB-C | 디바이스마트 | ✅ 구매 완료 |
| KIOXIA EXCERIA microSD 64GB | A1 등급 | 디바이스마트 | ✅ 구매 완료 |
| DSI 3.5인치 터치스크린 | Waveshare 권장 | 알리 | ⬜ 미구매 |
| LiPo 2000mAh (5mm) | 3.7V | 알리 | ⬜ 미구매 |

| GPIO 보호 캡 | 40핀 | 알리 | ⬜ 미구매 |

### 케이스 전략
- 1차: 케이스 없이 운영 (GPIO 캡 + 뒷면 아크릴판)
- 2차: 3D 프린팅 케이스 (킬 스위치/LED 장착 시)

## 보안 설계

### 위협 모델 우선순위
1. Tailscale 계정 탈취 → YubiKey + 강력한 비밀번호
2. SSH 공격 → 키 인증 전환 (예정)
3. 인터넷 직접 노출 금지 → 포트 포워딩 없음

### 현재 보안 상태
- OpenAI API 차단 (외부 전송 경로 없음) ✅
- Tailscale 암호화 통신 ✅
- SSH 키 인증 ⬜ (예정)

## 성능 실측 결과 (2026-06-20)
| 항목 | 수치 |
|---|---|
| eval rate (8B) | 2.10~2.29 tokens/s |
| 추론 중 최고 온도 | 64.2도 |
| 대기 온도 | 50~56도 |
| RAM 점유 (8B) | 6.3GB / 7.9GB |

→ RAM 빡빡 → 3B 기본 운영, 8B 필요시 수동 전환 전략 확정
