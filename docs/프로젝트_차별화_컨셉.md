# 프로젝트 차별화 컨셉 (Cyberdeck → Privacy-First Pocket Server)

## 한 줄 정의

> 외형 중심의 기존 사이버덱과 달리, **하드웨어 레벨 프라이버시 제어**와 **로컬 AI 자연어 인터페이스**를 갖추고, **휴대 단말 ↔ 개인 인트라넷 서버**를 오가는, **스마트폰 크기를 지향하는 초소형 프라이버시 우선 기기**

---

## 핵심 정체성 (3축)

| 축 | 내용 | 역할 |
|---|---|---|
| 프라이버시 | 오프라인 우선, 의도적 연결 | 기기의 철학 |
| AI 중심 제어 | AI가 인트라넷/기기의 인터페이스 | 사용자 조작 방식 |
| 듀얼 모드 | 휴대(배터리) ↔ 거치(유선 서버) | 기기의 두 상태 |

---

## 설계 최우선 제약: 초소형화

**목표: 기판(라즈베리파이 5, 85mm x 56mm)과 비슷한 크기. 스마트폰 폼팩터 지향.**

이 제약이 다른 모든 설계 결정을 지배함. 부품 선택 시 항상 "이게 폼팩터를 키우지 않는가?"를 먼저 검토.

### 초소형화에 유리한 선택

- **디스플레이**: HDMI 미니 모니터(5인치)는 라즈베리파이보다 큼 → DSI 연결 소형 디스플레이(3.5~4인치) 또는 보드 위에 적층(stack)하는 구조 검토
- **전원**: 외장 보조배터리 대신 보드에 적층되는 UPS HAT(18650 1셀) 또는 리튬폴리머 + 충전 모듈로 두께 최소화
- **입력**: 물리 키보드 내장은 폼팩터를 키움 → 보유 중인 블루투스 키보드를 외부 입력으로 분리(평소 휴대 시 무게/크기 절감)
- **쿨러**: 액티브 쿨러 높이가 두께에 영향 → 케이스 설계 시 쿨러 높이를 두께 기준으로 삼기
- **배선**: 커넥터/케이블이 차지하는 공간이 의외로 큼 → 납땜으로 직결하여 커넥터 부피 절감

### 초소형화의 트레이드오프 (인지 필요)

- 화면이 작아 긴 텍스트 입력/표시 불편 → AI 음성/단축 명령으로 보완
- 배터리 용량 제약 → 휴대 모드는 단시간, 장시간은 서버(거치) 모드 전제
- 발열 집중 → 작은 케이스일수록 방열 설계가 더 중요
- 적층 구조는 분해/수리가 어려움 → 조립 순서 신중히 설계

## 차별화 기능 (구현 우선순위)

### 기본 (1차 완성)

- 로컬 8B AI 구동 (Ollama)
- Tailscale 기반 개인 인트라넷
- 기기 간 안전한 파일 전송 (카톡 등 외부 경유 없음)

### +1 듀얼 모드 (먼저, 비교적 쉬움)

- 휴대 모드: 배터리 구동, AI 단말
- 서버 모드: 거치 시 유선(전원+이더넷) 연결 → 서버 스크립트 자동 실행
- 트리거: 거치 시 유선 연결 감지 → 모드 전환 스크립트

### +2 하드웨어 프라이버시 (납땜 작업)

- 물리 킬 스위치: WiFi/블루투스 전원을 하드웨어로 차단 (소프트웨어 우회 불가)
- 연결 상태 LED: 외부 네트워크 통신 중일 때만 점등 → "데이터 유출 중" 시각화
- 구현: GPIO + 릴레이/MOSFET (초소형 부품 선택으로 폼팩터 영향 최소화)

### +3 AI 명령 컨트롤러 (가장 어려움, 마지막)

- 자연어 → 인트라넷 명령 실행 (function calling 구조)
- 예: "어제 사진 노트북으로 보내줘", "외부 연결 다 꺼줘", "내 노트에서 OO 찾아줘"
- 8B 모델로 "정해진 명령 매칭" 수준 구현

---

## 포트폴리오 서술 포인트

- 단순 조립이 아닌 **명확한 설계 철학(프라이버시)** 기반 의사결정 강조
- 초소형화라는 **물리적 제약 하의 엔지니어링 트레이드오프** 경험 (전자공학 실무와 직결)
- 하드웨어(킬 스위치, 적층 설계) + 소프트웨어(AI, 인트라넷) **풀스택 통합** 역량
- "만들고 끝"이 아니라 **실제 일상에서 운영하는 시스템**
