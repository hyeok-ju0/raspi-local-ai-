# 프로젝트 차별화 컨셉

## 한 줄 정의
> 로컬 AI를 탑재한 주머니 사이즈의 휴대 단말. 다른 기기와 자료를 주고받을 때 카톡 등 외부 서비스 대신 안전한 직접 통로(Tailscale)를 통해 통신한다.

---

## 핵심 정체성
- **본질**: 주머니에 들어가는 로컬 AI 핸드헬드 단말
- **프라이버시 원칙**: 기본은 차단, 의도적으로만 열린다
- **인트라넷의 역할**: "서버"가 아니라 "안전한 자료 이동 통로"

---

## 설계 최우선 제약: 주머니 사이즈
- **평면**: 라즈베리파이 5 기판 크기(85×56mm) 기준 유지
- **두께 목표**: 40~50mm (재킷 주머니/가방 앞주머니 수납 가능)
- 스마트폰처럼 얇게는 비현실적 — USB-A 포트(15mm) + 액티브 쿨러 + 배터리 + 디스플레이 적층 시 물리적 한계. "주머니 수납 가능"을 1차 목표로 설정.

### 초소형화 설계 (확정)
- 디스플레이: **DSI 3.5인치 터치스크린**
- 전원: **LiPo 3000mAh + IP5305T 충방전 모듈** (UPS HAT 미사용 — 라즈베리파이 5 호환 불확실 및 두께 문제로 제외)
- 입력: 보유 중인 블루투스 키보드 활용 (내장 키보드 없음)
- 쿨러: 공식 액티브 쿨러 높이가 케이스 두께 기준
- 배선: 커넥터 대신 납땜 직결로 부피 절감

### 트레이드오프 (인지 필요)
- 화면 작음 → 긴 텍스트 입력/표시 불편 → AI 음성/단축 명령으로 보완 가능
- 배터리 제약 → 휴대 모드는 단시간 사용 전제
- 발열 집중 → 작은 케이스일수록 방열 설계 중요
- 적층 구조 → 분해/수리 어려움 → 조립 순서 신중히

---

## 운영 모델 (단계적 확장)

### 1차 완성 — 핸드헬드 단독 + 자료 주고받기
- 휴대 모드 단독 사용 (배터리 구동), 로컬 8B/3B AI를 핸드헬드 자체에서 사용, 외부 인터넷 연결 불필요
- 다른 기기(노트북, 폰, 태블릿)와 자료 주고받을 때만 Tailscale 활성화 → SCP/SFTP로 직접 전송, 카톡 등 외부 서비스 미경유 → 전송 끝나면 Tailscale 비활성화
- 24시간 서버 운영 없음: 필요할 때만 켜고 끔

### 2차 확장 — 거치 서버 모드
- 집에 거치대 마련 (USB-C 전원 + 이더넷), 거치 시 자동으로 서버 모드 활성화
- 외부에서 폰/노트북/태블릿으로 집 핸드헬드의 AI 접속
- 자동 모드 전환: 이더넷 케이블 연결 감지 = 거치됨 = 서버 모드 활성화 / 미연결 = 휴대 모드 = 외부 접속 차단
- 프라이버시 원칙 유지: 거치(의식적 행동) 없이는 외부 접속 차단, 우연한 외부 노출 없음

## 모델 운영 전략

### 듀얼 모델 (두 모델 디스크 보관, RAM은 한 번에 하나)
| 모델 | 역할 | 사용 빈도 |
|---|---|---|
| Llama 3.2 3B | 기본/일상 작업 | 자주 |
| Qwen3 8B | 복잡한 추론/긴 문서 | 가끔 |

- Ollama 자동 관리: 5분 미사용 시 RAM에서 자동 해제
- 디스크 점유: 합계 약 7GB (64GB SD카드에 여유 충분)
- 수동 전환: `ollama run` 명령어 또는 alias (`ai` / `ai-smart`)
- 운영 비율은 성능 실측(2026-06-20) 결과 기준 3B 기본 운영 + 8B 필요시 수동 전환으로 확정
- ⚠️ 동시 실행 금지: Qwen3 8B와 Llama 3.2 3B 동시 실행 시 RAM 초과로 SSH 연결 끊김 발생 → 모델 전환 시 반드시 `ollama stop <모델명>`으로 이전 모델 종료 후 전환

## 보안 설계

### 위협 모델 우선순위
1. **Tailscale 계정 탈취** — 가장 큰 단일 위협 → 강력한 비밀번호 + 2FA 필수
2. **SSH 비밀번호 무차별 대입** → SSH 키 인증으로 전환, Tailscale 인터페이스에서만 SSH 수신
3. **인터넷 직접 노출 금지** → 포트 포워딩 없이 Tailscale 안에서만 접근

### 부차적 위협 (인지만, 과한 대응 X)
- 분실/도난: 디스크 암호화는 불편함 대비 효용 낮음 → Tailscale 원격 차단 활용
- AI 로그 평문 저장: 라즈베리파이 자체 보안에 의존, 주기적 정리
- 같은 Wi-Fi 다른 기기: 집 Wi-Fi 비밀번호 강도로 충분

## 포트폴리오 서술 포인트
- 명확한 설계 철학: 프라이버시 우선 (기본 차단, 의도적 개방)
- 물리적 제약 하의 엔지니어링 트레이드오프: 주머니 사이즈 vs 성능/배터리/발열
- 하드웨어 + 소프트웨어 풀스택 통합: 회로(킬 스위치, LED) + 운영체제 + AI + 네트워크
- 단계적 확장 설계: 1차 완성 → 필요에 따라 +1/+2/+3 확장
- 실측 기반 의사결정: 모델 성능 측정 후 운영 방식 결정
- 실제 일상 운영: "만들고 끝"이 아닌 실사용 시스템
